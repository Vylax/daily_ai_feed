🧑‍🏫 Skill Up: Custom Tutorial - LangGraph basics

LangGraph allows you to build robust and stateful conversational agents as graphs, facilitating complex logic and error recovery. This tutorial demonstrates building a basic graph with two nodes and a conditional edge.

Objective

Create a LangGraph that transitions between two LLM-powered nodes based on the output of the first node.

Step 1: Install Dependencies

Install the necessary packages. Replace google-generativeai with the specific LLM provider package you are using if not using Google's generative AI models.

pip install langchain langchain-core langchain-community langgraph
pip install google-generativeai

Step 2: Configure API Keys

Set your API keys. This example uses the Google Gemini API. If using other APIs, adapt accordingly. Consider using environment variables for security.

import os
os.environ["GOOGLE_API_KEY"] = "YOUR_GOOGLE_API_KEY"

Step 3: Define the Nodes

Create two functions, each representing a node in the graph. Each node will use an LLM to process the input state. Ensure you install google-generativeai if you are using Google's Gemini model.

from langchain_core.prompts import ChatPromptTemplate, PromptTemplate
from langchain_google_genai import ChatGoogleGenerativeAI
from langgraph.graph import StateGraph, END
from typing import TypedDict, Dict, Any

class AgentState(TypedDict):
    messages: list[dict[str, Any]]
    steps: int

prompt_1 = PromptTemplate.from_template("Summarize the following text in one sentence: {text}")
prompt_2 = PromptTemplate.from_template("Suggest three follow up questions about this text: {text}")

model = ChatGoogleGenerativeAI(model="gemini-pro", convert_system_message_to_human=True)

def node_1(state: AgentState):
    prompt = prompt_1.invoke({"text": state["messages"][-1]["content"]})
    response = model.invoke(prompt)
    return {"messages": [{"role": "assistant", "content": response.content}]}

def node_2(state: AgentState):
    prompt = prompt_2.invoke({"text": state["messages"][-1]["content"]})
    response = model.invoke(prompt)
    return {"messages": [{"role": "assistant", "content": response.content}]}

Step 4: Define the Graph

Create the `StateGraph` and define the nodes, edges, and entry point.

from langgraph.graph import StateGraph

builder = StateGraph(AgentState)
builder.add_node("node_1", node_1)
builder.add_node("node_2", node_2)

# Define conditional edge.  We'll use a simple condition - step count.
def should_continue(state: AgentState):
    if state["steps"] &lt; 2: # Run at most twice
        return "node_2"
    else:
        return "end"

builder.add_conditional_edges(
    start_key="node_1",
    condition=should_continue,
    conditional_edge_mapping={
        "node_2": "node_2",
        "end": END
    }
)
builder.add_edge("node_2", "node_1")

builder.set_entry_point("node_1")
graph = builder.compile()

Step 5: Run the Graph

Run the graph with an initial state.

inputs = {"messages": [{"role": "user", "content": "The quick brown fox jumps over the lazy dog."}], "steps": 0}
chain = graph.with_config({"recursion_limit": 100}) # Increase recusion limit for longer chains
result = chain.invoke(inputs)
print(result)

Key Considerations for Real-World Use

  • API Key Management: Never hardcode API keys in your code. Use environment variables or a secrets management system.
  • Error Handling: Implement robust error handling within each node and in the graph execution. Use `try...except` blocks to catch exceptions and retry failed operations.
  • State Management: Choose an appropriate state management strategy (e.g., in-memory, Redis, database) based on your application's requirements.
  • Logging and Monitoring: Implement comprehensive logging to track the execution of the graph and monitor performance.
  • Rate Limiting and Concurrency: Be mindful of API rate limits and implement appropriate concurrency controls to prevent overloading external services.
  • Scaling: Consider using distributed computing frameworks (e.g., Dask, Ray) to scale your LangGraph applications for high throughput.
