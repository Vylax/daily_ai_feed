🧑‍🏫 Skill Up: Custom Tutorial - LangGraph basics

Objective: Build a simple LangGraph that makes a decision based on user input and either answers directly or performs a search.

Core Concepts:

  • Nodes: Functions or language models that process information.
  • Edges: Connections that route information between nodes, determined by conditions or logic.
  • Graph State: Data passed between nodes as the graph executes.

Prerequisites:

  • langchain_core==0.1.25
  • langgraph==0.0.34
  • langchain==0.1.4
  • langchain_openai==0.0.5
  • beautifulsoup4
  • requests
  • typing
  • os
pip install langchain_core==0.1.25 langgraph==0.0.34 langchain==0.1.4 langchain_openai==0.0.5 beautifulsoup4 requests typing

Step-by-Step Implementation:

  1. Setup: Import necessary libraries and configure your OpenAI API key.

    ```python import os from typing import TypedDict, Literal, Dict, Any from langchain_core.runnables import chain from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder from langchain_core.messages import BaseMessage from langchain_openai import ChatOpenAI from langgraph.graph import StateGraph, END from langchain.chains import create_extraction_chain from langchain.pydantic_v1 import BaseModel, Field

    Set your OpenAI API key (or use environment variables)

    os.environ["OPENAI_API_KEY"] = "YOUR_OPENAI_API_KEY" # Replace with your key! `` *Explanation:* Imports the necessary modules fromlangchain,langchain_core,langgraph, and sets up your OpenAI API key. **Important:** Replace"YOUR_OPENAI_API_KEY"` with your actual key.

  2. Define the Graph State: Define a TypedDict to represent the state that will be passed between nodes.

    ```python class GraphState(TypedDict): """ Represents the state of our graph.

    input: User input string
messages: List of messages
knowledge: str
answer: str
"""
input: str
messages: list[BaseMessage]
knowledge: str
answer: str

    ```

    Explanation: This defines the data structure that will be available to each node in the graph. input will hold the user's question, messages is used for conversational history, knowledge contains results from the search, and answer stores the final response.

  3. Define Nodes: Tool Decision (Router): Define a function to decide whether to search or answer directly.

    ```python model = ChatOpenAI(temperature=0) # or gpt-4

    def router(state: GraphState): """ Determines whether to search for knowledge or respond directly. """ messages = state['messages'] input = state['input']

    prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant that can either answer questions directly or search for information."
     "If you think you can answer the question directly, do so."
     "If you think you need to search for information first, respond with 'SEARCH'."
     "Otherwise, answer the question directly."),
    ("user", "{input}")
])

response = prompt | model
result = response.invoke({"input": input, "messages": messages}).content

if "SEARCH" in result.upper():
    return "search"
else:
    return "answer"

    ```

    Explanation: This node acts as a router. It takes the current graph state, formulates a prompt for the LLM (using OpenAI's ChatOpenAI), and decides whether to search ("search" node) or answer directly ("answer" node) based on the LLM's response.

  4. Define Nodes: Knowledge Search (Tool): Define a simplified search function (replace with a more robust search if needed).

    ```python from bs4 import BeautifulSoup import requests

    def web_search(query: str) -> str: """Searches the web and returns the results.""" url = f"https://google.com/search?q={query}" headers = { "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/58.0.3029.110 Safari/537.3" } try: response = requests.get(url, headers=headers) response.raise_for_status() # Raise HTTPError for bad responses (4XX, 5XX) soup = BeautifulSoup(response.text, 'html.parser') results = [result.get_text() for result in soup.find_all('div', class_='tF2Cxc')] # Adapt selector as needed return "\n".join(results[:3]) # Return the top 3 results except requests.exceptions.RequestException as e: return f"Search failed: {e}"

    def search(state: GraphState): """Searches for relevant knowledge.""" query = state['input'] knowledge = web_search(query) return {"knowledge": knowledge} ```

    Explanation: This node simulates a search tool. It uses requests and BeautifulSoup4 to scrape Google search results (very basic!). It returns the extracted text as knowledge.

  5. Define Nodes: Answer Directly (LLM): Define a function to answer the question directly using the LLM.

    ```python def answer(state: GraphState): """Answers the question directly, potentially using knowledge.""" messages = state['messages'] input = state['input'] knowledge = state['knowledge']

    prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant. Answer the question based on the provided context. If the context is empty, just answer the question directly."),
    MessagesPlaceholder(variable_name="messages"),
    ("user", "{input}"),
    ("system", "Context: {knowledge}"),
])

response = prompt | model
result = response.invoke({"input": input, "messages": messages, "knowledge": knowledge}).content
return {"answer": result}

    ```

    Explanation: This node uses the LLM to formulate an answer. It considers the user's input, any existing conversation history (messages), and any retrieved knowledge (from the search node, if executed).

  6. Create the Graph: Assemble the graph using StateGraph.

    ```python workflow = StateGraph(GraphState)

    workflow.add_node("router", router) workflow.add_node("search", search) workflow.add_node("answer", answer)

    workflow.add_edge("router", "search", condition=lambda state: state['knowledge'] == "") workflow.add_conditional_edges( "router", lambda state: "answer", {"search": "search", "answer": "answer"} )

    workflow.add_edge("search", "answer") # Changed to a simple edge

    workflow.set_entry_point("router") workflow.set_finish_point("answer")

    graph = workflow.compile() ```

    Explanation: This code defines the graph's structure. It adds the router, search, and answer nodes. It defines conditional edges so the flow starts at the router, then it determines whether to execute the search or go directly to the answer node, finally ending at the answer node in all cases. It then compiles the graph into an executable object.

    ```python

    Example usage

    if name == "main": inputs = {"input": "What is the capital of France?", "messages": [], "knowledge": "", "answer": ""} result = graph.invoke(inputs) print(f"Answer: {result['answer']}")

    inputs = {"input": "What is the latest news about LangGraph?", "messages": [], "knowledge": "", "answer": ""}
result = graph.invoke(inputs)
print(f"Answer: {result['answer']}")

    ```

    Explanation: This shows how to invoke the graph. We provide a starting state with the user's input. The graph.invoke() method executes the graph and returns the final state, which contains the answer.
