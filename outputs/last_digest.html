🧑‍🏫 Skill Up: Custom Tutorial - LangGraph basics

This tutorial provides a concise introduction to LangGraph, focusing on building a simple graph-based agent workflow. We'll build a simple agent that decides whether to search the internet or answer a question directly.

Objective

To create a basic LangGraph workflow with a decision node and two possible actions.

Step 1: Installation and Setup

First, install the required LangChain and LangGraph packages.

# Install necessary libraries
!pip install -U langchain langchain-google-vertexai langgraph tavily-python

Next, set up your environment. You'll need a Google Cloud project and API keys. Configure your Google Cloud credentials.

Step 2: Define the Components

We'll create a simple Language Model (LLM) and a search tool. For brevity, we'll use the Google Vertex AI model and Tavily Search. Remember to obtain and set the required API keys.

import os
from langchain_google_vertexai import VertexAI
from langchain.tools import tool
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_core.runnables import chain

# Set up environment variables (replace with your actual keys)
os.environ["TAVILY_API_KEY"] = "YOUR_TAVILY_API_KEY"  # Replace with your Tavily API Key

# Initialize LLM
llm = VertexAI(model_name="gemini-1.5-pro-001", temperature=0.1)

# Define a tool
@tool
def search_internet(query: str) -> str:
    """Useful for searching the internet for current events or general knowledge."""
    from tavily import TavilyClient

    tavily = TavilyClient(api_key=os.environ["TAVILY_API_KEY"])
    results = tavily.search(query=query)
    return results

tools = [search_internet]

# Define prompt
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant. Answer the user's questions."),
    MessagesPlaceholder(variable_name="messages"),
    MessagesPlaceholder(variable_name="agent_scratchpad")
])

# Define agent
def get_response(messages, agent_scratchpad):
    formatted_prompt = prompt.format_messages(messages=messages, agent_scratchpad=agent_scratchpad)
    return llm.invoke(formatted_prompt)

def parse_response(response):
    if "I need to use a tool" in response:
        return "search"
    else:
        return "answer"

agent = chain({
    "messages": lambda x: x["messages"],
    "agent_scratchpad": lambda x: x["agent_scratchpad"]
}, get_response) | parse_response

tool_map = {tool.name: tool for tool in tools}
def call_tool(tool_name, messages, agent_scratchpad):
    input_text = agent_scratchpad[-1].content
    return tool_map[tool_name].invoke({"query": input_text})

Step 3: Construct the LangGraph

Now, let's define the graph. We'll use three nodes: agent, search, and answer. We'll also define a state object to hold the agent's memory.

from langgraph.graph import StateGraph, MessageGraph
from typing import TypedDict, List, Dict, Any
from langchain_core.messages import BaseMessage

class AgentState(TypedDict):
    messages: List[BaseMessage]
    agent_scratchpad: List[BaseMessage]
    steps: List[List[str]]

workflow = StateGraph(AgentState)

# Define nodes
workflow.add_node("agent", agent)
workflow.add_node("search", call_tool)
workflow.add_node("answer", lambda x: x["agent_scratchpad"][-1].content) # this will be the last content message

# Define edges
workflow.set_entry_point("agent")
workflow.add_conditional_edges(
    "agent",
    lambda x: "search" if x == "search" else "answer",
    {
        "search": "search",
        "answer": "answer"
    }
)

# Build graph
graph = workflow.compile()

Step 4: Run the Graph

Finally, let's run the graph with an example question.

from langchain_core.messages import HumanMessage, AIMessage

inputs = {"messages": [HumanMessage(content="What is the current temperature in London?")], "agent_scratchpad": []}
result = graph.invoke(inputs)

print(result)

Key Considerations for Production

  • API Key Management: Use secure methods (e.g., environment variables, secret management services like Google Cloud Secret Manager) for storing and accessing API keys.
  • Error Handling: Implement robust error handling and retry mechanisms within each node of the graph. Use LangChain's callback system for monitoring and logging.
  • State Management: Consider the scalability and persistence of the agent's state, especially for long-running conversations. Options include in-memory storage (for prototyping), Redis, or a dedicated vector database for more complex scenarios.
  • Observability: Integrate with tracing tools like LangSmith or other monitoring solutions to track the performance and behavior of your LangGraph workflows. Pay close attention to latency and cost per request.
  • Security: Sanitize user inputs to prevent prompt injection attacks. Be mindful of the data privacy implications of your agent's actions.
